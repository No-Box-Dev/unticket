Hello, world.

If you can read this rendered (not as source), the Unticket Specs proxy is wired up end-to-end.

What this proves

  • The repo No-Box-Dev/unticket is readable via the Unticket GitHub App.
  • The Pages Function at /specs-content/ proxies the bytes with the right Content-Type (so the browser renders the HTML instead of showing source).
  • Relative-linked assets in the same folder resolve under the same proxy URL — this page's ./style.css being loaded is the proof.
  • The ut_session cookie authenticates the proxy without an Authorization header (browsers don't send those for sub-resource loads).

Configure unticket

In Settings → Specs source:

RepoNo-Box-Dev/unticket
Root folderspecs

The Specs tab will then list hello-world as one spec card. Click into it and click Open on this index.html to land back here.

Add more demo specs

Drop another top-level folder next to hello-world/ in specs/ and it'll show up as its own card. Files inside a spec folder render inline if they're Markdown, or open here via the proxy if they're HTML.
